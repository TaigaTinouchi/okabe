# ADR-0006: クライアント認証は静的 Bearer トークン1本

- Status: Accepted (2026-07-13)

## Decision

認証は環境変数で与える**静的 Bearer トークン1本**。WebSocket ハンドシェイクと REST 全域で検証する。
HTTPS 終端はリバースプロキシ（Caddy）に任せる。

## Context / 理由

サーバーは公開エンドポイントになるため無認証は論外。一方でユーザーは自分1人・クライアントは自作アプリのみ。
OAuth / JWT / セッション管理は「守るべき他人」がいて初めて意味を持つ装置であり、
ここでは複雑さがそのまま攻撃面と保守負担になる。

- トークンは十分長いランダム値（`openssl rand -base64 32` 等）を `.env` に置く
- 漏洩時のローテーションは「.env を書き換えて再起動、アプリ側の設定を更新」の2手（README に手順記載）
- レートリミット等は前段の Caddy に足せる

## Consequences

- 「作り込まない」判断として明文化し、マルチユーザー化の予定がないことを前提に置く
- トークンがログに出ないよう、認証ミドルウェアはヘッダ値をログしない
